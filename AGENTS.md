# NixOS Dotfiles

## Critical Non-Obvious Behaviors

### Rime Input Method Data (`rime-sync/nixos-ysun/`)

These files are auto-generated by the input method and change frequently. Always commit them as-is alongside the main work of the turn; do not stop to ask whether they should be included, and do not mention them in the commit message unless the user explicitly asks. Conflicts are expected and usually trivial: keep the newer timestamp.

### Nushell Proxy Wrapper

`claude-proxy` wraps Claude Code and injects proxy environment variables from `~/.config/claude/proxy.nuon`. If network access looks broken, check `home/shell/nushell/config.nu`.

## Key Constraints

- The repo's shell workflow is Nushell-first. Prefer Nushell syntax when writing or updating repo commands and examples.
- When executing Nushell snippets through Codex tools, do not rely on the tool's shell selection alone. Invoke Nushell explicitly as `nu -c '...'`, otherwise the command may still be interpreted by `/bin/sh`.
- Secrets must go through sops-nix. See `docs/secret-management.md`.
- Changes are verified manually by the user after rebuild.
- System rebuild command:
  ```nushell
  sudo nixos-rebuild switch --flake .#homePC
  ```
